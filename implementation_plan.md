# 學生進度追蹤與臨床教學對應開發計畫

本計畫涵蓋實習生 28 週進度排程、成績整合分析，以及 AI 輔助個人化學習計畫 (ILP) 的實作藍圖。

## 📅 進度基準與核心規則
- **循環起點**：一律以當年度「7月的第 1 個星期一」作為 Week 1 (第1週)。
- **時間長度**：統一為 **28 週**。
- **最後兩週 (W27, W28)**：預設配置為「手動自選週」。
- **複選科別**：考量到實習排程彈性，**每週允許配置複數個站別**（小站）。

---

## 🚀 實作階段規劃

### 🔴 Phase 1: 核心進度引擎與管理員排程介面 (已完成 ✅)
1. 實作 `get_current_intern_week(date)` 計算 28 週時間軸。
2. 建立 Google Sheets `學生進度排程` 並透過 API `/api/admin/schedule` 進行讀寫。
3. 實作管理員端 `schedule_manager.html` (支援多選標籤、W27/W28 預設自選)。
4. 實作學生儀表板的「本週進度卡片」。

### 🟡 Phase 2: 臨床教學評分自動映射 (進度分析)
1. **紀錄比對與儀表板**
   - 系統依據「該週分配的複數站別」，檢查學生是否達成對應的 OPA 評估記錄次數。
   - 於介面中給予「進度達標燈號」（🟢 達標 / 🔴 未達標 / ⚪ 未開始）。

### 🔵 Phase 3: 管理員 Excel 總表匯出功能
1. **成績統計匯出**
   - 於管理員頁面實作統整所有學生「實習時數」、「OPA各站成績」、「AKS相關評估」以及「進度符合率」的大表。
   - 透過後端 Python 生成 Excel 或 CSV 檔案，提供管理員一鍵下載。

### 🟣 Phase 4: AKS 爬蟲擴充 & 學生 PDF 匯出 (待收斂清單)
1. **AKS 資料提取**
   - 擴充 `ceep_scraper.py` 取出 Attitude, Knowledge, Skills 存入 `教學紀錄_AKS` 表單。
2. **學生履歷 PDF 匯出**
   - 整合 `html2pdf.js`，將頁面上所有圖表與文字打包成 PDF 證明。

### 🟢 Phase 5: AI 輔助 ILP (Individualized Learning Plan) 雙軌生成
1. **學員數據統整 (Data Aggregation)**
   - 系統定期彙整每位學員的 **量性指標** (OPA/AKS 分數) 與 **質性指標** (教師回饋評語)。
2. **雙 AI 模型串接比較 (A/B Testing)**
   - 透過 API 介接**兩種不同的 AI 模型** (例如可透過環境變數配置 Model A 與 Model B 的 API 端點/金鑰)。
   - 同時將相同的學員資料與固定 Prompt 丟給兩個模型處理。
   - 於後台介面並排展示兩個 AI 產出的「客觀評核報告與建議」，供教師比較或採納。
3. **週期性產出 (Weekly Routine)**
   - 觸發節點設定為 **每星期 (Weekly)** 產出一份。
   - 系統將根據當週學員的表現變化與累積數據，每週自動生成專屬的 ILP 報告供追蹤。
